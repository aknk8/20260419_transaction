## 1. はじめに

本システムは、見積・受注・発注・請求・入金・支払を一元管理する取引管理システムです。  
業務フローに沿って各伝票を作成・承認することで、取引の追跡性確保と転記作業の削減を実現します。

### 1.1 業務の基本的な流れ

```
見積作成 → 見積承認 → 受注作成 → 受注承認 → 発注起票 → 発注承認
 → 納品・検収 → 請求作成 → 請求承認 → 請求書送付 → 入金消込
                         ↘ 支払依頼 → 支払承認 → 支払登録
```

### 1.2 初期リリースの通知手段について

現在のバージョンでは、すべての通知は **画面内の通知一覧（S-14）** のみで行われます。メール通知は将来対応予定です。

## 2. 利用者区分と権限の概要

### 2.1 利用者区分

| 区分 | 説明 |
|------|------|
| **システム管理者** | ユーザ管理・権限管理・マスタ管理・全体監査を行う |
| **一般ユーザ** | 所属部門・役職・個別権限に応じた業務利用 |

### 2.2 部門別の主な利用範囲

| 部門 | 主な利用画面 | 主な業務 |
|------|------------|---------|
| **営業部門** | 案件、見積、受注、発注（参照） | 見積作成・承認依頼、受注作成・承認依頼 |
| **営業事務部門** | 見積、受注、請求 | 見積作成補助、請求書作成 |
| **購買部門** | 発注、納品・検収、支払依頼 | 発注起票、納品検収登録、支払依頼作成 |
| **経理部門** | 請求、入金、支払、レポート | 入金消込、支払登録、集計 |
| **管理部門** | 受注（契約手続き）、承認 | 受注後の契約手続き完了操作 |
| **システム管理者** | 全画面 | ユーザ・マスタ管理、監査 |

### 2.3 権限の3層構造

権限は以下の3層の組み合わせで決定します。

1. **利用者区分**（システム管理者 / 一般ユーザ）
2. **所属部門・役職**（部長、課長、担当者 など）
3. **ユーザ個別権限**（特定ユーザへの追加付与または制限）

### 2.4 操作権限の種類

| 権限 | 内容 |
|------|------|
| 参照 | 一覧・詳細の閲覧 |
| 作成 | 新規伝票の作成・下書き保存 |
| 更新 | 下書き状態の伝票の編集 |
| 承認 | 承認・却下操作 |
| 出力 | PDF・CSV 出力 |
| 削除 | 伝票の取消・削除 |

> **注意：** 権限がない操作のボタンは画面上に表示されません。操作できない理由が不明な場合はシステム管理者に問い合わせてください。

## 3. S-01 ログイン画面

### 3.1 概要

システムを利用するための認証を行います。すべての利用者が使用します。

### 3.2 操作手順

1. ブラウザで取引管理システムのURL にアクセスする
2. **ユーザID** と **パスワード** を入力する
3. **ログイン** ボタンをクリックする
4. 認証成功後、ダッシュボード（S-02）へ遷移する

### 3.3 認証失敗時の動作

| 状態 | 表示内容 |
|------|---------|
| IDまたはパスワードが誤り | エラーメッセージ表示 |
| 連続認証失敗 | 失敗回数を記録（アカウントロックの場合は通知） |
| アカウントロック状態 | ロック中の旨を通知 |
| パスワード変更要求 | パスワード変更を促す通知 |

### 3.4 ログイン後の画面制御

ログイン後に表示されるメニューや操作ボタンは、**所属部門・役職・個別権限** に基づいて自動制御されます。

## 4. S-02 ダッシュボード

### 4.1 概要

ログイン直後に表示される画面です。担当業務の進捗状況を一目で確認できます。

### 4.2 表示内容

| 指標 | 内容 | 対象ユーザ |
|------|------|---------|
| **承認待ち件数** | 自分が承認者として割り当てられている伝票数（伝票種別ごと） | 承認権限保有者 |
| **未請求件数** | 請求対象フラグが立っているが請求書未作成の受注数 | 営業・経理部門 |
| **未収件数** | 送付済みまたは一部入金状態の請求書数 | 経理部門 |
| **未払件数** | 承認済みで支払未完了の支払依頼数 | 購買・経理部門 |

> **注意：** 承認待ち件数は「自分が申請した伝票」は含まれません。自分が承認者として割り当てられている件数のみ表示されます。件数ゼロの伝票種別はグレーアウト表示されます。

### 4.3 ダッシュボードからの遷移

| 操作 | 遷移先 |
|------|--------|
| 承認待ち件数（各種別）をクリック | S-12 承認一覧（対象種別でフィルター済み） |
| 各指標をクリック | 対応する一覧画面 |

### 4.4 権限による表示制御

自分の所属部門・役職・個別権限に関係する指標のみ表示されます。例えば購買部門には「未払件数」が表示されますが、経理部門専用の指標は表示されない場合があります。

## 6. S-04 見積一覧・登録・詳細

### 6.1 概要

顧客への見積書を作成・管理する画面です。承認後に受注へ転換します。

**主な利用者：** 営業部門、営業事務部門

### 6.2 一覧画面でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 一覧表示・検索 | 参照 | ステータス・顧客・案件・担当者・日付などで絞り込み |
| CSV出力 | 出力 | 一覧データのCSV出力 |
| 新規作成 | 作成 | 新規見積を作成 |

### 6.3 見積登録・編集

見積コードは自動採番されます（例：`QUO-00001`）。

| 項目 | 説明 |
|------|------|
| 案件 | 紐付ける案件を選択（任意） |
| 顧客 | 見積先顧客を選択 |
| 見積タイトル | 件名を入力 |
| 発行日・有効期限 | 日付を入力 |
| 明細行 | 商品・数量・単価・値引・税率を入力（行の追加・削除が可能） |
| 備考 | 自由記入欄 |

**金額の自動計算：**  
各明細行の金額 ＝ 単価 × 数量 − 値引 + 消費税（税率に基づき自動計算）

### 6.4 見積の状態遷移

```
下書き → 承認依頼中 → 承認済み（→ 受注作成へ）
                    ↘ 却下（→ 下書きに戻る）
下書き → 失注
下書き → 取消
```

| 操作 | 権限 | 条件 |
|------|------|------|
| 保存（下書き） | 作成・更新 | いつでも可能 |
| 承認依頼 | 作成 | 下書き状態のとき |
| 承認 | 承認 | 承認依頼中のとき（承認権限保有者のみ） |
| 却下 | 承認 | 承認依頼中のとき（却下コメントが必須） |
| 複製 | 作成 | 任意のステータスから新しい下書きを作成 |
| 改版 | 作成 | 旧版を保持したまま版数を上げた新しい下書きを作成 |
| 失注登録 | 更新 | 失注理由を入力して記録 |
| 取消 | 更新 | 下書き状態のとき |
| PDF出力 | 出力 | 承認済み状態のとき（承認前は出力制限可） |

### 6.5 社長決裁が必要な場合

以下の条件に該当する場合、**社長の承認** が追加で必要になります（条件の閾値はシステム設定で変更可能）。

- 見積金額合計が設定した上限額を超える場合
- 利益率が設定した下限率を下回る場合
- 上記の OR 条件または AND 条件の組み合わせ

**承認ルート（例）：**  
申請者（営業担当）→ 営業部長 → （条件該当時）社長

### 6.6 承認履歴の確認

詳細画面下部の **承認状況エリア** で以下を確認できます。

- 現在の承認ステップ（例：「承認依頼中（ステップ 1/2：営業部長 確認待ち）」）
- 承認履歴テーブル（操作日時・操作者・操作種別・コメント）

**承認履歴は申請者・承認者の両方が閲覧できます。**

### 6.7 できないこと（権限なし）

- 参照権限がない利用者はアクセスできません
- 承認権限がない場合は「承認」「却下」ボタンが表示されません
- 出力権限がない場合はPDF出力ボタンが表示されません

## 7. S-05 受注一覧・詳細

### 7.1 概要

顧客からの正式な注文を管理する画面です。承認済み見積から作成します。

**主な利用者：** 営業部門、営業事務部門

### 7.2 受注の作成

受注は **承認済み見積** から作成します。見積の主要情報（顧客・明細・金額など）が自動引き継ぎされます。  
受注コードは自動採番されます（例：`ORD-00001`）。

### 7.3 受注の状態遷移

```
受注済み → 承認依頼中 → 承認済み → 契約手続き済
                      ↘ 却下（→ 下書きに戻る）
受注済み → 取消
```

| ステータス | 意味 |
|-----------|------|
| 受注済み | 見積から転換されたが承認前の状態 |
| 承認依頼中 | 営業部長への承認依頼中 |
| 承認済み | 営業部長が承認した状態 |
| 契約手続き済 | 管理部が契約手続き完了を記録した状態 |
| 却下 | 却下されて下書きに戻る |
| 取消 | 受注を取り消した状態 |

### 7.4 受注承認依頼の条件（必須チェック）

以下がすべて満たされないと承認依頼を送信できません。

1. **添付ファイルが1件以上あること**（契約書または注文書）
2. **見積申請が紐付けられていること**
3. **受注金額と見積金額が一致していること**（合計金額で判定）

> **注意：** 下書き保存段階では添付ファイルなしでも保存できますが、承認依頼送信時には必須です。

### 7.5 一覧・詳細でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 一覧表示・検索 | 参照 | ステータス・顧客・案件・日付などで絞り込み |
| 受注作成 | 作成 | 承認済み見積から受注を作成 |
| 添付ファイル追加・削除 | 更新 | 契約書・注文書を添付 |
| 承認依頼送信 | 作成 | 必須チェック通過後に依頼送信 |
| 承認・却下 | 承認 | 承認権限保有者（営業部長）のみ |
| 契約手続き完了 | 更新 | 管理部長のみ操作可能 |
| 請求対象化 | 更新 | 受注を請求対象としてフラグ設定 |
| 発注起票 | 作成 | 受注から発注を起票（購買部門） |

### 7.6 承認ルート

申請者（営業担当）→ 営業部長

### 7.7 できないこと（権限なし）

- 管理部門以外の利用者は「契約手続き完了」を操作できません
- 承認権限がない場合は「承認」「却下」ボタンが表示されません

## 8. S-06 発注一覧・登録・詳細

### 8.1 概要

仕入先・外注先への発注を管理する画面です。

**主な利用者：** 購買部門、営業部門（参照）、システム管理者

### 8.2 発注の作成

発注コードは自動採番されます（例：`POD-00001`）。

**作成方法：**
- 案件から起票する
- 受注明細から起票する（受注情報が自動引き継ぎ）

仕入先が複数の場合は **仕入先別に発注を分割** して作成できます。

### 8.3 発注の状態遷移

```
下書き → 承認依頼中 → 承認済み・発注待ち → 発注済み → 納品済
                    ↘ 却下（→ 下書きに戻る）
下書き → 取消
```

### 8.4 一覧・詳細でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 一覧表示・検索 | 参照 | ステータス・仕入先・日付などで絞り込み |
| 発注起票 | 作成 | 案件または受注明細から発注を新規作成 |
| 明細編集 | 更新 | 品名・数量・単価・税率を編集（下書き状態のみ） |
| 承認依頼送信 | 作成 | 内容確認後に承認依頼 |
| 承認・却下 | 承認 | 承認権限保有者のみ |
| 発注書PDF出力 | 出力 | 承認後に発注書を出力して仕入先へ送付 |
| 納品予定日登録 | 更新 | 仕入先から納期回答を受けてシステムに登録 |
| 発注残確認 | 参照 | 発注残数量・発注残金額を確認 |
| CSV出力 | 出力 | 一覧データのCSV出力 |

**発注残の計算式：**  
- 発注残数量 ＝ 発注数量 − 検収合格数量
- 発注残金額 ＝ 発注金額 − （検収合格数量 × 発注単価）

> 検収不合格数量は発注残として継続計上されます。

### 8.5 できないこと（権限なし）

- 参照権限がない利用者はこの画面にアクセスできません（メニューに表示されません）
- 承認権限がない場合は「承認」「却下」ボタンが表示されません
- 出力権限がない場合は発注書PDF出力ボタンが表示されません

## 9. S-07 納品・検収一覧・登録

### 9.1 概要

仕入先からの納品実績と検収結果を記録する画面です。発注一覧（S-06）とは独立したナビゲーション項目として提供されます。

**主な利用者：** 購買部門

### 9.2 一覧画面でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 一覧表示 | 参照 | 納品実績を一覧表示 |
| 絞り込み | 参照 | 発注コード・ステータス・納品日で絞り込み |
| 発注詳細へ遷移 | 参照 | 該当行から S-06 発注詳細へ遷移 |
| 納品実績登録 | 作成 | 新規納品実績を登録 |

### 9.3 納品実績の登録

| 項目 | 説明 |
|------|------|
| 発注明細または受注明細の指定 | 対象明細を選択 |
| 納品日 | 実際に納品を受けた日付 |
| 数量 | 今回納品された数量 |
| 検収結果 | 「検収済」または「検収NG」を選択 |
| 備考 | 特記事項があれば入力 |

> **一部納品・一部検収に対応：** すべての明細を一度に検収する必要はありません。

### 9.4 ステータスの自動遷移

すべての明細の検収が完了すると、該当発注は自動的に「**納品済**」ステータスに変更されます。

### 9.5 できないこと（権限なし）

- 購買部門以外の利用者は登録操作ができない場合があります
- 参照のみ権限の場合は新規登録ができません

## 10. S-08 請求一覧・登録・詳細

### 10.1 概要

顧客への請求書を作成・管理する画面です。受注の「請求対象化」操作を起点として作成します。

**主な利用者：** 営業事務部門、経理部門

### 10.2 請求書の作成

請求コードは自動採番されます（例：`INV-00001`）。

**作成の起点：**
- 受注が「請求対象」フラグを持っている状態から請求書を作成
- 受注情報（顧客・明細・金額など）が自動引き継ぎされます
- 複数受注をまとめた **合算請求** にも対応

### 10.3 請求の状態遷移

```
下書き → 承認依頼中 → 承認済み → 確定 → 送付済 → 一部入金 → 消込済み
                    ↘ 却下（→ 下書きに戻る）
確定済み → 取消（取消または訂正伝票で対応）
```

### 10.4 一覧・詳細でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 一覧表示・検索 | 参照 | ステータス・顧客・日付・締日などで絞り込み |
| 請求書作成 | 作成 | 請求対象受注から請求書を作成 |
| 明細編集 | 更新 | 品名・数量・単価を編集（下書きのみ） |
| 承認依頼送信 | 作成 | 内容確認後に依頼送信 |
| 承認・却下 | 承認 | 承認権限保有者のみ |
| 請求書PDF出力 | 出力 | 確定後に請求書を出力して顧客へ発行 |
| 請求残高確認 | 参照 | 入金済み額と未収残高を確認 |
| CSV出力 | 出力 | 一覧データのCSV出力 |

### 10.5 承認ルート

申請者（営業担当）→ 営業部長 → 管理部担当者 → 管理部長

> 管理部長承認後、管理部担当者が請求書を出力して顧客へ発行します。

### 10.6 消費税の計算

複数受注の合算請求の場合、消費税は**課税合計に対して一括計算**されます（明細ごとの税額合計ではありません）。

### 10.7 できないこと（権限なし）

- 確定後の請求書は直接変更できません（取消または訂正伝票で対応）
- 承認権限がない場合は「承認」「却下」ボタンが表示されません
- 出力権限がない場合はPDF出力ボタンが表示されません

## 11. S-09 入金登録

### 11.1 概要

顧客からの入金を登録し、請求書と消込を行う画面です。

**主な利用者：** 経理部門

### 11.2 入金登録の操作

入金コードは自動採番されます（例：`RCP-00001`）。

| 項目 | 説明 |
|------|------|
| 対象請求書 | 消込対象の請求書を選択 |
| 入金日 | 実際に入金を確認した日付 |
| 入金額 | 入金された金額 |
| 手数料 | 振込手数料などがあれば入力 |
| 差額理由 | 差額が発生した場合の理由を入力 |

### 11.3 消込の処理

| 状況 | 処理 |
|------|------|
| 請求金額と入金額が一致 | 消込済みとなる |
| 一部入金（不足） | 残額を未収残高として継続管理 |
| 過入金（超過） | 差額を「差額あり」状態で記録し調整伝票を処理 |

**消込状態：**
- 未消込
- 消込済み
- 差額あり

### 11.4 できないこと（権限なし）

- 経理部門以外はこの画面にアクセスできない場合があります
- 消込操作には専用の権限が必要です

### 12.5 できないこと（権限なし）

## 13. S-11 マスタ管理

### 13.1 概要

システムで使用するマスタデータを管理する画面です。

**主な利用者：** システム管理者のみ

> **重要：** マスタ更新は **システム管理者権限保有者のみ** が実施できます。一般ユーザはこの画面にアクセスできません。

### 13.2 管理できるマスタ

| マスタ種別 | 主な管理項目 |
|-----------|------------|
| **顧客マスタ** | 顧客名・住所・担当者・締日・支払サイト・請求先 |
| **仕入先マスタ** | 仕入先名・住所・担当者・支払条件 |
| **商品・サービスマスタ** | 商品名・標準単価・税区分・利用可否 |
| **ユーザマスタ** | ユーザ名・所属部門・役職・利用可否 |
| **部門マスタ** | 部門名・部門コード |
| **税率マスタ** | 税区分・税率 |
| **支払条件マスタ** | 支払条件の定義 |

### 13.3 各マスタでできること

| 操作 | 説明 |
|------|------|
| 一覧検索・詳細参照 | マスタ種別ごとに一覧で検索・詳細を確認 |
| 新規登録 | 新しいマスタデータを登録 |
| 更新 | 既存データの修正 |
| 無効化 | データを論理削除（物理削除はしない） |
| 履歴確認 | 更新履歴を確認 |

### 13.4 ユーザ権限の設定

ユーザマスタで以下を設定できます。

- **所属部門・役職** の設定
- **個別権限** の追加付与または制限（参照・作成・更新・削除・承認・出力）
- 権限変更後は次回ログイン時に反映されます

### 13.5 システム設定

以下のシステム設定を管理できます。

| 設定項目 | 説明 |
|---------|------|
| 社長決裁条件（閾値） | 見積金額の上限額・利益率の下限値（数値のみ変更可） |
| 承認ルート | 伝票種別・ステップ番号・承認者ユーザIDの登録 |
| 承認滞留判定日数 | N-04通知を出すまでの日数（初期値：3日） |

> **注意：** 承認条件の「種類」（ANDかORか、金額か利益率かなど）を変更する場合はシステム修正が必要です。「閾値（数値）」と「承認者」はシステム設定で変更できます。

## 14. S-12 承認一覧

### 14.1 概要

自分に割り当てられた承認依頼を一覧表示し、承認・却下操作を行う画面です。

**主な利用者：** 承認権限保有者

### 14.2 一覧画面でできること

| 操作 | 権限 | 説明 |
|------|------|------|
| 承認依頼一覧表示 | 承認 | 自分宛の承認依頼を一覧で確認 |
| 絞り込み | 承認 | 伝票種別・申請者・申請日・承認状態で絞り込み |
| 伝票詳細へ遷移 | 参照 | 行クリックで対象伝票の詳細画面へ遷移 |

### 14.3 承認・却下操作

1. 一覧から対象の承認依頼行をクリックして伝票詳細画面へ遷移する
2. 詳細画面上部の「**承認する**」または「**却下**」ボタンをクリックする
3. 却下の場合は **コメント入力欄が展開され、コメント入力が必須** となる
4. 承認の場合はコメント入力は任意
5. 操作完了後は承認一覧画面に自動で戻る

> **注意：** 自分が申請した伝票はこの画面に表示されません。承認できるのは「自分が承認者として割り当てられている伝票」のみです。

### 14.4 承認時のルール

| 場合 | 動作 |
|------|------|
| 同一ステップに複数の承認者がいる場合 | **全員の承認**をもって当該ステップが完了（AND条件） |
| 承認が完了した場合 | 申請者へ通知が届く（N-02） |
| 却下した場合 | 申請者へ却下コメント付き通知が届く（N-03） |
