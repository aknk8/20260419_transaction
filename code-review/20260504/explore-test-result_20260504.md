### BUG-01: 新規見積フォームの自動採番が "1" を返す

- **再現手順**: admin でログイン → 見積メニュー → 新規登録ボタン → コードフィールドを確認
- **期待値**: `QUO-00006`（既存5件の次番号）
- **実際値**: `1`
- **影響**: 新規見積が登録できない、または既存データとの重複・不整合が発生する可能性
- **推定原因**: `src/quotation.js` の採番ロジックが連番整数を返しており、`QUO-XXXXX` フォーマットへの変換が欠落している疑い

### BUG-03: 見積ステータスフィルターの不一致

- **問題**: フィルター選択肢に「却下」が含まれ、「取消」が含まれない
- **要件定義§7.3の定義**: `下書き / 承認依頼中 / 承認済み / 失注 / 取消`（「却下」は存在しない）
- **影響**: ユーザーが「却下」ステータスで絞り込み操作を行うと空結果になる。「取消」ステータムのデータをフィルタリングできない
- **該当箇所**: `app.js` 行1666付近のステータスフィルター `<option>` 列挙

- **問題**: 発注一覧テーブルで一部行のステータスが "承認済・発注待ち" と表示される
- **推定原因**: 発注ステータスの定義値に存在しない文字列、またはテーブルセルへのデータマッピング時に undefined がそのまま文字列化されている可能性
